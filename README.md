1️⃣ GitHub는 왜 쓰는가?
"GitHub를 쓰는 진짜 이유는 복합적인 생각의 흐름을 증명 하기 위해서 사용합니다."
• 신뢰의 지표: 나는 이만큼 압니다라는 말보다 6개월간 꾸준히 쌓인 커밋 내역이 더 강력한 증거가 되어 증명이 됩니다.
• 협업의 표준: 혼자 만드는 것이 아니라 타인의 코드를 읽고 내 생각을 섞어 더 큰 시스템을 만드는 소통의 장소입니다.
• 복기: 과거의 내가 왜 이런 코드를 짰는지 기록함으로써 미래의 내가 더 나은 의사결정을 하도록 돕는 보조장치 역할을 합니다.

2️⃣ Repository / Commit / Issue / README의 의미
• Repositor: 프로젝트마다 환경과 설정이 꼬이지 않도록 소스 코드와 모든 변경 이력을 따로 떼어 관리하는 독립된 작업 공간입니다.
• Commit: 어제는 왜 됐지?라는 상황에 대비함과 동시에 의미 있는 코드 변화가 생길 때마다 논리적 근거를 남겨두는 과정의 누적 기록입니다.
• Issue: 무엇을 먼저 고치고 만들지 헤매지 않도록 해결해야 할 문제나 추가할 기능을 미리 정의해 두는 전략적 할 일 목록입니다.
• README: 코드를 다 뜯어보지 않아도 이 프로젝트가 왜 존재하고 어떻게 실행하는지 한눈에 알려주는 가이드북입니다.

3️⃣ 좋은 README란 무엇인가?
"제가 생각하는 좋은 README는 단순히 기술들을 나열하는 것이 아닌 이걸 왜 만들었으며 이 과정에서 내가 무엇을 배웠는가 명확히 보이는 체계적인 글입니다."
• 기술보다 이유가 우선: 단순히 자바를 썼다가 아니라 자바의 이 개념이 헷갈려서 이걸 확인해 보려고 만들었다는 목적이 드러나야 합니다.
• 상대방의 시간 아끼며 효율적이게: 내 저장소에 처음 들어온 사람이 코드를 다 뜯어보지 않아도 1분 안에 아, 이런 거구나라고 이해할 수 있게 만드는 배려가 담고 빠른 이해와 단축시키는 시간의 일석이조의 효율을 만드는 것입니다.
• 성장의 기록: 완벽한 결과물보다 어떤 에러를 만났고 그걸 어떻게 풀었는지 적어둔 시간의 과정이 담긴 README가 진짜 좋은 기록이라고 생각합니다.

4️⃣ 코드가 없어도 GitHub를 쓰는 방법
"코드 없이도 성장 매뉴얼과 공부 일기장으로 사용을 합니다."
• Issue를 활용하기: 질문 메모장의 역할로 공부하다 막히는 게 생기면 이슈에 적어둔 후 나중에 답을 찾았을 때 댓글을 달아 Close하는 과정을 통해 공부 로그로 사용이 가능합니다.
• 나만의 학습 가이드 만들기: 공부하면서 본 좋은 블로그나 유튜브 링크를 README에 정리한 뒤 날짜 및 시간을 적어 둔 뒤 계획에 맞춰 나중에 다시 찾기 편하게 나만의 학습 로드맵을 만드는 것입니다. 
• 성장 시스템 시각화: 매일 공부한 내용을 체크박스로 표시하며 루틴을 관리합니다. 코드가 없어도 내가 매일 무언가를 하고 있다는 과정 속에서 성장하고 있다는 성실함의 증거를 남기는 용도로 활용합니다.

* 마지막으로 
- 왜 개발을 배우려는가
다양한 직무 현장에서 "안정적인 결과는 정교한 시스템에서 나온다"는 본질을 깨닫게 되었으며 이제는 마케팅,서비스적 감각을 기술적 전문성으로 치환하여 AI 시대에 문제를 정의하고 시스템으로 해결하는 대체 불가능한 개발자가 되기 위해서 개발을 배워 취업을 하려고 원티드랩에 신청을 하였습니다.

- 내가 기대하는 개발자의 모습
단순히 AI가 뱉어주는 코드를 복사하는 단순 업무를 하는 사람이 아닌 데이터의 흐름과 예외 케이스를 직접 설계하고 제어할 줄 아는 개발자가 되고 싶습니다. 또한 구체적으로 단순히 기능만 돌아가게 만드는 것이 목적이 아니라 AI를 도구로 활용하되 그 결과물의 논리적 결함을 검증하고 구조적 안정성을 책임지는 설계자가 되고싶습니다, 어떤 복잡한 비즈니스 로직이라도 예측 가능한 범위 안에서 관리하여 동료들이 제 코드를 믿고 자신의 로직을 쌓아 올릴 수 있는 단단한 시스템의 기준이 되는 것이 제가 기대하는 모습의 개발자입니다.

- 한 달 뒤 바뀌고 싶은 점
Java와 객체지향 원리를 내재화하여 데이터 흐름과 메모리 구조를 설계할 수 있는 기초 체력을 기르고 에러 앞에서 당황하지 않고 "왜 안 되는지"를 논리적으로 추적하여 스스로 답을 도출하는 과정을 내재화한 모습을 기대하고 있으며 되도록 노력할 것입니다.
